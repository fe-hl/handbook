Maven 核心知识优化版 ​

1. 为什么使用 Maven ​

  • Jar包管理革命
    • 存储优化：统一仓库管理，避免重复下载（本地仓库+远程仓库）
    • 下载优化：镜像加速（阿里云）+ 依赖传递自动解析
  • 项目构建标准化
    • 7步标准化流程：清理 → 编译 → 测试 → 报告 → 打包 → 安装 → 部署
    • 跨平台支持：Java编写，但支持多种语言项目（C/PHP/Python）

2. 安装配置精要 ​

  • 环境变量配置

    bash
    # 系统变量
JAVA_HOME = JDK安装路径
MAVEN_HOME = Maven解压路径
Path += %MAVEN_HOME%\bin

  • 验证命令
    mvn -v → 显示Apache Maven版本及JDK信息

  • 核心配置（conf/settings.xml）

    xml
    <!-- 本地仓库路径 -->
<localRepository>D:/maven_repo</localRepository>

<!-- 阿里云镜像 -->
<mirror>
  <id>aliyun</id>
  <mirrorOf>*</mirrorOf>
  <url>https://maven.aliyun.com/repository/public</url>
</mirror>

<!-- JDK17配置 -->
<profile>
  <id>jdk-17</id>
  <activation>
    <activeByDefault>true</activeByDefault>
  </activation>
  <properties>
    <maven.compiler.source>17</maven.compiler.source>
  </properties>
</profile>

3. 项目结构规范 ​

工程类型目录结构关键文件
普通Java工程src/main/java + src/test/javapom.xml
Web工程增加 src/main/webapp/WEB-INFweb.xml + pom.xml

4. 核心机制解析 ​

🔄 依赖传递机制 ​

mermaid
graph LR
A[项目A] --> B[jar-B]
B --> C[jar-C]
A --> C[自动传递]
  • 传递条件：作用域必须为compile
  • 冲突解决原则：
    1. 最短路径优先
    2. 先声明优先

🛡️ 依赖作用域（Scope） ​

Scope主代码测试代码服务器用例
compile✅✅✅Spring Core
provided✅✅❌Servlet API
test❌✅❌JUnit

🚫 依赖排除 ​

xml
<dependency>
  <groupId>com.example</groupId>
  <artifactId>projectB</artifactId>
  <exclusions>
    <exclusion>
      <groupId>com.fasterxml.jackson.core</groupId>
      <artifactId>jackson-databind</artifactId>
    </exclusion>
  </exclusions>
</dependency>

5. 高级特性实践 ​

🧬 继承管理 ​

父工程配置：

xml
<dependencyManagement>
  <dependencies>
    <dependency>
      <groupId>org.springframework</groupId>
      <artifactId>spring-context</artifactId>
      <version>6.1.15</version>
    </dependency>
  </dependencies>
</dependencyManagement>

子工程引用（⚠️ 不写版本号）：

xml
<dependencies>
  <dependency>
    <groupId>org.springframework</groupId>
    <artifactId>spring-context</artifactId>
  </dependency>
</dependencies>

🧩 聚合工程 ​

xml
<!-- 父pom.xml -->
<modules>
  <module>order-service</module>
  <module>user-service</module>
  <module>web-app</module>
</modules>
  • 构建优势：mvn clean install 一键构建所有子模块

6. 构建生命周期 ​

周期命令作用常用组合
Cleanmvn clean删除target目录mvn clean package
Defaultmvn compile编译主代码mvn clean install
mvn test运行单元测试
mvn package生成jar/war包
mvn install部署到本地仓库
Sitemvn site生成项目报告

💡 插件是生命周期的最小执行单元

7. 实战配置模板 ​

📦 打包优化配置 ​

xml
<build>
  <!-- 自定义输出名称 -->
  <finalName>myapp</finalName>
  
  <!-- 资源文件包含规则 -->
  <resources>
    <resource>
      <directory>src/main/resources</directory>
      <includes>
        <include>**/*.xml</include>
        <include>**/*.properties</include>
      </includes>
    </resource>
  </resources>
  
  <!-- JDK17兼容插件 -->
  <plugins>
    <plugin>
      <groupId>org.apache.maven.plugins</groupId>
      <artifactId>maven-war-plugin</artifactId>
      <version>3.3.2</version> <!-- 最新稳定版 -->
    </plugin>
  </plugins>
</build>
